# SSH Manager v1.3.0 - 核心体验优化

## 版本说明

本次更新重点优化了核心用户体验，实现了所有连接和文件传输的直接操作功能。

---

## ✨ 新增/优化功能

### 1. ✅ 批量连接 - 即时创建多个终端标签

**改进内容：**
- 批量连接成功后，立即在终端会话中创建多个标签页
- 无需等待，所有成功连接的服务器立即可用
- 失败的服务器不影响成功连接的创建
- 清晰的失败报告显示失败原因

**使用流程：**
1. 在连接库中勾选多台服务器
2. 点击"批量连接"
3. 系统逐个尝试连接
4. **立即看到**：所有成功的服务器在终端会话中创建标签
5. **失败显示**：失败列表和原因显示在弹窗中
6. **直接使用**：切换到终端会话即可操作任何一台服务器

**技术实现：**
- 使用Electron IPC事件系统
- 连接成功后立即发送事件到终端组件
- 终端组件异步创建会话和标签页
- 失败不影响成功的连接

### 2. ✅ 单台连接 - 跳转后立即显示终端

**改进内容：**
- 点击连接后立即跳转到终端会话
- 自动激活对应的终端窗口
- 无需在终端页面重新点击服务器

**使用流程：**
1. 在连接库中点击"连接"按钮
2. 自动跳转到"终端会话"页面
3. **立即看到**：对应的终端窗口已激活，可直接输入命令

### 3. ✅ 文件拖拽上传 - 无需先选中

**改进内容：**
- 直接从本地文件浏览器拖拽文件到远端面板
- 无需先在列表中选中文件
- 按当前显示的远端路径上传
- 传完自动刷新右侧远端列表

**使用方式：**
1. 确保已连接到SSH服务器
2. 在文件浏览中找到本地文件
3. **直接拖拽**：从本地文件拖到右侧远端面板
4. 自动上传到当前显示的远端目录
5. 右侧列表自动刷新，显示新上传的文件

**适用场景：**
- 快速上传配置文件
- 上传代码文件到服务器
- 上传压缩包或安装包

### 4. ✅ 文件拖拽下载 - 无需先选中

**改进内容：**
- 直接从远端文件列表拖拽到左侧本地面板
- 无需先在列表中选中文件
- 下载到左侧当前显示的本地目录
- 传完自动刷新左侧本地列表

**使用方式：**
1. 确保已连接到SSH服务器
2. 在右侧远端文件列表中找到文件
3. **直接拖拽**：从远端文件拖到左侧本地面板
4. 自动下载到当前显示的本地目录
5. 左侧列表自动刷新，显示新下载的文件

**适用场景：**
- 快速下载日志文件
- 拉取服务器配置文件
- 下载编译产物到本地

## 🎯 完整操作指南

### 批量连接操作

```
步骤1: 在连接库中勾选多台服务器（使用复选框）
步骤2: 点击"批量连接"按钮
步骤3: 等待连接完成（系统自动处理）
步骤4: 查看结果弹窗：
       • 成功 X 台 - 已创建终端标签
       • 失败 X 台 - 显示失败原因
步骤5: 点击确定自动跳转到终端会话
步骤6: 在终端会话中看到所有成功连接的标签
步骤7: 点击标签即可切换服务器，直接输入命令
```

### 文件拖拽传输

**上传文件到服务器：**
```
步骤1: 进入"文件浏览"页面
步骤2: 连接SSH服务器
步骤3: 在左侧本地面板中找到文件
步骤4: 直接拖拽文件到右侧远端面板
步骤5: 等待上传完成（可拖拽多个文件）
步骤6: 右侧列表自动刷新显示新文件
```

**从服务器下载文件：**
```
步骤1: 进入"文件浏览"页面
步骤2: 确保已连接SSH服务器
步骤3: 在右侧远端面板中找到文件
步骤4: 直接拖拽文件到左侧本地面板
步骤5: 等待下载完成
步骤6: 左侧列表自动刷新显示新文件
```

### 单台快速连接

```
步骤1: 在连接库中找到服务器
步骤2: 点击"连接"按钮
步骤3: 自动跳转到终端会话
步骤4: 终端窗口已激活，可直接输入命令
```

## 💡 使用技巧

### 高效运维工作流

1. **日常连接**：使用批量连接一次打开多台服务器
2. **文件传输**：拖拽文件即可上传/下载，无需点击按钮
3. **命令执行**：在终端中直接输入命令，无需切换页面
4. **监控多机**：使用分屏功能同时查看多台服务器

### 文件管理技巧

1. **快速上传**：直接从Windows文件管理器拖拽到应用
2. **批量下载**：选中多个文件一起下载
3. **目录同步**：在两侧面板切换不同目录进行对比
4. **新建目录**：点击"新建目录"按钮创建文件夹

---

## ⚠️ 注意事项

1. **网络要求**：确保SSH连接稳定，文件传输时避免断网
2. **权限问题**：确保SSH账号有相应的读写权限
3. **大文件**：超过100MB的文件建议使用压缩或分批传输
4. **字符编码**：确保文件名不包含特殊字符避免传输失败
